## 1. 工作流总览

### 1.1 双图并行架构

项目存在两套并行的 StateGraph：

| Graph | 节点数 | 触发条件 | 典型场景 |
|-------|--------|---------|---------|
| **Research Graph** | 7 节点 | `source_type=research` | 调研主题 → 综述报告 |
| **Report Graph** | 11 节点 | `source_type=arxiv/pdf` | 单篇论文 → 带引用的 Markdown 报告 |

```
┌──────────────────────────────────────────────────────────────────┐
│                         Research Graph (7 节点)                    │
│                                                                  │
│  ┌─────────┐   ┌──────────────┐   ┌───────┐   ┌────────┐        │
│  │clarify │──→│ search_plan  │──→│search │──→│extract │        │

│       │ needs_followup?                  │          │            │

│                                       review_passed? ▼            │

│                                   └─────────────────┘             │
└──────────────────────────────────────────────────────────────────┘
```

## 2. 各节点代码流程详解

### 2.1 Clarify Node — 需求澄清

**文件**：`src/research/graph/nodes/clarify.py`

**职责**：接收用户原始研究主题，LLM 生成结构化 ResearchBrief。

```python
def run_clarify_node(state: dict) -> dict:
    """clarify_node: 将用户模糊主题 → 结构化 ResearchBrief"""
    topic = state.get("raw_input", "")
    settings = get_settings()

    # 使用 reasoning LLM 深度理解用户意图
    llm = build_reason_llm(settings, timeout_s=45, temperature=0.1)

    # 构建澄清 prompt（包含 field definition）
    messages = build_clarify_messages(topic)
    response = llm.invoke(messages)

    # Pydantic 严格解析
    brief = ClarifyOutput.model_validate_json(response.content)

    # 决策：是否需要追问
    if brief.needs_followup:
        return {"brief": brief, "_route": END}
    return {"brief": brief, "_route": "search_plan"}
```

**ClarifyOutput Schema**（`src/research/prompts/clarify_prompt.py`）：

```python
class ClarifyOutput(BaseModel):
    topic: str                          # 精炼后的研究主题
    sub_questions: list[str]           # 子问题列表
    desired_output: str                 # 期望输出格式
    inclusion_criteria: list[str]       # 纳入/排除标准
    time_range: str | None              # 时间范围（如 2020-01 ~ 2025-12）
    ambiguities: list[str]              # 仍存在的模糊点
    needs_followup: bool                # 是否需要追问
```

**路由决策**（`builder.py`）：

```python
def _route_after_clarify(state: dict) -> str:
    brief = state.get("brief")
    if not brief:
        return END
    if brief.get("needs_followup", False):
        return END  # 需追问时终止，等待用户反馈
    return "search_plan"
```

**为什么需要 Clarify**：用户输入往往是模糊的（如"AI Agent 在医疗领域的发展"），直接检索会产生大量不相关结果。ClarifyAgent 的作用：
- 分解为具体子问题（"医疗影像诊断"、"药物发现"、"患者随访"等）
- 设定时间范围（近5年？全时间段？）
- 明确纳入标准（顶会论文？中文论文？开源工作？）

### 2.2 SearchPlan Node — 检索规划

**文件**：`src/research/graph/nodes/search_plan.py`

**职责**：基于 ResearchBrief，生成多组查询策略（覆盖核心方法/数据集/最新进展）。

```python
def run_search_plan_node(state: dict) -> dict:
    """search_plan_node: 生成 SearchPlan（query_groups / plan_goal / warnings）"""
    brief: ResearchBrief = state["brief"]
    settings = get_settings()

    # 支持两种模式：heuristic（规则生成）和 agent（LLM 自主生成）
    if policy_type == "agent":
        llm = build_reason_llm(settings, timeout_s=45)
        # Agent 模式：LLM 自主生成多组查询
        plan = _agent_generate_plan(llm, brief)
    else:
        # Heuristic 模式：规则 + LLM 辅助
        plan = _heuristic_plan(brief)

    return {"search_plan": plan, "_route": "search"}
```

### 2.3 Search Node — 三源并行检索

**文件**：`src/research/graph/nodes/search.py`

**职责**：SearXNG + arXiv API + DeepXiv 三源并行检索，去重后前 50 篇入库。

```python
def search_node(state: dict) -> dict:
    """三源并行检索 + 去重 + ingest"""
    plan: SearchPlan = state["search_plan"]
    brief: ResearchBrief = state["brief"]

    all_candidates = []
    queries = [q for g in plan.query_groups for q in g.queries]

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # 并行搜索：ThreadPoolExecutor(max_workers=3)
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    with ThreadPoolExecutor(max_workers=3) as executor:
        futures = {
            executor.submit(_searxng_search, queries): "searxng",
            executor.submit(_arxiv_api_search, queries): "arxiv",
            executor.submit(_deepxiv_search, queries): "deepxiv",
        }
        for future in as_completed(futures):
            source = futures[future]
            try:
                results = future.result(timeout=30)
                # 优先级：arXiv > DeepXiv > SearXNG
                all_candidates.extend(results)
            except Exception as e:
                logger.warning(f"[search_node] {source} failed: {e}")

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # 去重合并（按 arxiv_id 精确去重）
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    deduped = _dedupe_by_arxiv_id(all_candidates)

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # 批量 enrich：补充 arXiv metadata（作者、年份等）
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    enriched = enrich_search_results_with_arxiv(deduped[:100])

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # ingest 前 50 篇到 PostgreSQL ChunkStore
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    _ingest_paper_candidates(enriched[:50])

    return {"rag_result": RagResult(paper_candidates=enriched[:50]), "_route": "extract"}
```

**三源并行设计考量**：

| 源 | 优势 | 劣势 |
|----|------|------|
| **SearXNG** | 多引擎聚合，无 API key，广度召回 | 可能返回非学术来源 |
| **arXiv API** | 权威来源，元数据完整 | 查询语法限制 |
| **DeepXiv** | 学术专注，趋势论文 | 覆盖率有限 |

**去重策略**：基于 `arxiv_id + url` 精确去重，优先级 arXiv > DeepXiv > SearXNG。

### 2.4 Extract Node — 批量 LLM 抽取

**文件**：`src/research/graph/nodes/extract.py`

**职责**：对每篇候选论文，通过 LLM 抽取 PaperCard（结构化摘要）。

```python
def extract_node(state: dict) -> dict:
    """批量 LLM 抽取 PaperCard（3 篇/批并行）"""
    rag_result: RagResult = state["rag_result"]
    candidates = rag_result.paper_candidates
    settings = get_settings()

    paper_cards = []
    # 批量并行：3 篇/批，减少 LLM 调用次数
    for i in range(0, len(candidates), BATCH_SIZE):
        batch = candidates[i : i + BATCH_SIZE]
        results = _extract_batch(batch, settings)
        paper_cards.extend(results)

    return {"paper_cards": paper_cards, "_route": "draft"}
```

**PaperCard Schema**（`src/corpus/models.py`）：

```python
class PaperCard(BaseModel):
    arxiv_id: str
    title: str
    authors: list[str]
    abstract: str
    methods: str | None
    datasets: list[str] | None
    limitations: str | None
    year: int | None
    venue: str | None
```

**LLM 抽取 Prompt 策略**：单篇论文 abstract 作为输入，LLM 结构化抽取 fields。避免了正则表达式的脆弱性。

### 2.5 Draft Node — 结构化报告生成

**文件**：`src/research/graph/nodes/draft.py`

**职责**：基于 ResearchBrief + PaperCards，LLM 生成 10-section 结构化 DraftReport。

```python
def draft_node(state: dict) -> dict:
    """生成 10-section 结构化 DraftReport"""
    brief: ResearchBrief = state["brief"]
    paper_cards: list[PaperCard] = state["paper_cards"]

    # 渲染 paper cards 为文本（供 LLM 阅读）
    cards_text = _render_cards(paper_cards[:20])  # 最多 20 篇

    # 构建带 JSON schema 约束的 system prompt
    system_prompt = _build_draft_system_prompt(brief)

    messages = [
        SystemMessage(content=system_prompt),
        HumanMessage(content=f"以下是论文卡片：\n{cards_text}"),
    ]

    llm = build_reason_llm(settings, timeout_s=240, max_tokens=8192)
    response = llm.invoke(messages)

    try:
        draft = DraftReport.model_validate_json(response.content)
    except Exception:
        # Fallback：基于 template 的粗糙生成
        draft = _fallback_draft(paper_cards[:20], brief)

    # 注入 citation content（用 paper_cards 的 abstract 填充 fetched_content）
    draft = _inject_citation_content(draft, paper_cards)

    return {"draft_report": draft, "_route": "review"}
```

**DraftReport Schema**（`src/models/report.py`）：

```python
class DraftReport(BaseModel):
    title: str
    sections: dict[str, str]    # {section_name: content}
    claims: list[Claim]
    citations: list[Citation]

class Claim(BaseModel):
    id: str
    text: str                  # 具体可验证的声明
    citation_labels: list[str]  # 引用的标签（如 ["[1]", "[2]"]）
    supports: list[ClaimSupport]
    overall_status: Literal["supported", "partial", "unsupported", "abstained"]

class Citation(BaseModel):
    label: str
    url: str
    reason: str                 # 为什么引用这篇
    source_tier: SourceTier     # A/B/C/D 四级权威度
    reachable: bool
    fetched_content: str | None
```

### 2.6 Review Node — 引用验证 + 质量评审

**文件**：`src/research/graph/nodes/review.py`

**职责**：ground_draft_report（resolve_citations → verify_claims → format_output）+ ReviewerService 四类检查。

```python
def review_node(state: dict) -> dict:
    """引用验证 + 质量评审"""
    draft_report: DraftReport = state["draft_report"]

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # Step 1: Citation Resolution（URL 可达性 + source tier）
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    resolved = resolve_citations(draft_report.citations)

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # Step 2: Claim Verification（LLM Judge）
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    verified = verify_claims(draft_report.claims, resolved)

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # Step 3: Format Output（应用 abstention policy）
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    final_report = format_output(verified, resolved)

    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    # Step 4: ReviewerService 四类质量检查
    # ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
    reviewer = ReviewerService()
    feedback = reviewer.review(
        paper_cards=state["paper_cards"],
        draft_report=final_report,
        brief=state["brief"],
    )

    review_passed = len([i for i in feedback.issues if i.severity == "blocker"]) == 0

    return {
        "resolved_report": resolved,
        "verified_report": verified,
        "final_report": final_report,
        "review_feedback": feedback,
        "review_passed": review_passed,
        "_route": "persist_artifacts" if review_passed else END,
    }
```

**路由决策**：

```python
def _route_after_review(state: dict) -> str:
    if state.get("review_passed"):
        return "persist_artifacts"
    return END  # review 失败则终止（可扩展 revise 节点）
```

### 2.7 Persist Artifacts Node — 持久化

**文件**：`src/research/graph/nodes/persist_artifacts.py`

**职责**：将所有中间产物（Brief、Plan、Cards、Report、Feedback）持久化到 PostgreSQL + workspace 文件系统。

```python
def persist_artifacts_node(state: dict) -> dict:
    """持久化所有中间产物"""
    workspace_id = f"ws_{state['task_id'][:12]}"

    # PostgreSQL Snapshot
    upsert_task_snapshot(
        task_id=state["task_id"],
        stage="completed",
        state_snapshot=state,  # 完整状态 JSONB
    )

    # ChunkStore：已入库的 paper chunks 持久化
    persist_chunks_to_db(state.get("rag_result", {}).get("chunks", []))

    # Workspace 文件系统
    workspace_path = Path(f"workspaces/{workspace_id}")
    (workspace_path / "report.md").write_text(state["final_report"].markdown)
    (workspace_path / "feedback.json").write_text(state["review_feedback"].model_dump_json())

    return {"workspace_id": workspace_id}
```

## 3. 条件降级机制（Degradation Mode）

**文件**：`src/research/graph/builder.py`

```python
def build_research_graph(emitter, degradation_mode: str = "normal"):
    """
    degradation_mode: normal | limited | safe_abort

    - normal: 完整 7 节点，所有 LLM 调用
    - limited: 跳过 review，减少 LLM 调用
    - safe_abort: 只做 clarify 和 search_plan，直接返回
    """
```

**降级策略场景**：
- `limited`：当 LLM API 不可用时，跳过 Review 直接生成报告
- `safe_abort`：当检索 API 不可用时，只做需求澄清然后返回

## 5. 优点总结

| 维度 | 实现方式 | 优势 |
|------|---------|------|
| **可观测性** | instrument_node 包装每个节点 | 计时、token、warnings 全链路追踪 |
| **可恢复性** | TaskSnapshot JSONB 持久化 | 任意节点崩溃后可重跑 |
| **可配置降级** | degradation_mode 参数 | 恶劣环境下仍可提供服务 |
| **可扩展性** | conditional edges + 节点注册 | 新增节点只需 add_node + 写路由函数 |
| **状态类型安全** | TypedDict + Pydantic v2 | 运行时类型校验，提前发现 schema 不匹配 |
