### 1.2 Supervisor 编排架构

**文件**：`src/research/agents/supervisor.py`

```python
# Supervisor 的 LEGACY / V2 切换逻辑
class AgentSupervisor:
    def _get_backend_mode(self, node_name: str) -> NodeBackendMode:
        canonical = self.normalize_node_name(node_name)
        node_mode = self.config.node_backends.mode_for(canonical)
        execution_mode = self.config.execution_mode

        if execution_mode == ExecutionMode.LEGACY:
            return NodeBackendMode.LEGACY   # → 直接调用 graph node 函数
        if execution_mode == ExecutionMode.V2 and self._has_v2_backend(canonical):
            return NodeBackendMode.V2         # → 调用 V2 Agent 类
        return NodeBackendMode.LEGACY

    def collaborate(self, initial_state: dict) -> dict:
        """主循环：按 CANONICAL_NODE_ORDER 顺序执行"""
        for node in CANONICAL_NODE_ORDER:
            mode = self._get_backend_mode(node)

            if mode == NodeBackendMode.LEGACY:
                result = self._run_legacy_node(node, state)  # → graph node
            else:
                result = self._run_v2_agent(node, state)    # → Agent 类

            state.update(result)
            self._record_handoff(node, mode, state)
```

## 2. 主流 Multi-Agent 工程范式对比

### 2.1 四大主流范式

| 范式 | 核心思想 | 代表工作 | 适用场景 |
|------|---------|---------|---------|
| **Supervisor-Worker** | 中央调度 + 专业 Worker | LangGraph Supervisor Pattern | 任务可分解、结构固定 |
| **Plan-and-Execute** | 规划 → 执行分离 | Plan-and-Act (WebArena) | 长时、多步骤、需重规划 |
| **Reasoning-via-Artifacts** | 中间工件驱动推理 | 本项目 AnalystAgent | 需要结构化综合的场景 |
| **Reflexion / MAR** | 自反思 + 多视角批评 | MAR (EMNLP 2024) | 需要自我纠正的场景 |

### 2.2 各范式详解与论文链接

#### 范式 A：Supervisor-Worker（中央调度）

**核心思想**：一个中央 Supervisor 负责任务分解和结果汇总，Worker 负责具体执行。

**本项目对应实现**：`AgentSupervisor.collaborate()` — 按 `CANONICAL_NODE_ORDER` 顺序调度各节点。

**论文参考**：
- [The Supervisor-Worker Agent Pattern: Foundations, Applications, and Future Trends](https://atoms.dev/insights/the-supervisor-worker-agent-pattern-foundations-applications-and-future-trends/a9e79aba927347d1bdd47c7bcd5d38da)
- [LangGraph Supervisor Pattern — DeepWiki](https://deepwiki.com/langchain-ai/langgraphjs/4.1-supervisor-pattern)

**优点**：简单直观、易调试、易保证输出格式一致性
**缺点**：中央单点，Worker 无自主决策能力

---

#### �式 B：Plan-and-Execute（规划-执行分离）

**核心思想**：Planner 生成高层次计划，Executor 逐步执行，支持动态重规划。

**本项目对应实现**：`SearchPlanAgent`（生成 SearchPlan） + `search_node`（执行搜索）。

**论文参考**：
- [Plan-and-Act: Improving Planning of Agents for Long-Horizon Tasks (2025)](https://arxiv.org/html/2503.09572v3)
  - 效果：WebArena-Lite 57.58%，WebVoyager 81.36%
- [AIME: Adaptive Multi-Agent Planning](https://arxiv.org/pdf/2602.16873) — 动态重规划框架

**本项目的局限**：
- Plan 生成后直接执行，无动态重规划
- `needs_followup=True` 时直接 END，没有重规划回退

---

#### 范式 C：Reasoning-via-Artifacts（工件驱动推理）

**核心思想**：生成中间结构化工件（分类矩阵、大纲），再基于工件做分析性写作。

**本项目对应实现**：`AnalystAgent` 的 RVA 流水线：

```python
# analyst_agent.py — RVA LangGraph
workflow.add_edge(START, "seed_reasoning_state")
workflow.add_edge("seed_reasoning_state", "build_structured_cards")
workflow.add_edge("structured_cards", "build_comparison_matrix")
workflow.add_edge("comparison_matrix", "build_outline")
workflow.add_edge("outline", "build_report_draft")
workflow.add_edge("build_report_draft", "verify_and_finalize")
```

**优势**：避免单次 LLM 调用承担所有任务，通过中间工件实现渐进式综合

**论文参考**：
- [Reasoning via Artifacts — OpenAI Research](https://arxiv.org/abs/2306.06085)

---

#### 范式 D：Reflexion / MAR（多智能体自反思）

**核心思想**：Actor 生成动作 → 多视角 Critic 评估 → 反射更新记忆。

**本项目对应实现**：`ReviewerAgent`（ReviewerService 四类检查）。

**论文参考**：
- [MAR: Multi-Agent Reflexion Improves Reasoning Abilities in LLMs (EMNLP 2024)](https://huggingface.co/papers/2512.20845)
  - 效果：HumanEval Pass@1 从 76.4% → 82.6%（+6.2pts）
  - HotPotQA Exact-Match 从 44% → 47%（+3pts）
  - 解决了单 Agent Reflexion 的确认偏见问题
- [DEVIL'S ADVOCATE: Anticipatory Reflection](https://aclanthology.org/2024.findings-emnlp.53.pdf)
  - WebArena 23.5% 成功率，45% 更少 trial

### 3.2 SearchPlanAgent — 检索规划

**文件**：`src/research/agents/search_plan_agent.py`

```python
class SearchPlanAgent:
    """检索规划 Agent：支持 heuristic 和 agent 两种模式"""
    def run(self, brief: ResearchBrief) -> SearchPlanResult:
        if self.mode == "heuristic":
            return self._heuristic_plan(brief)
        else:
            return self._agent_plan(brief)  # LLM 自主生成

    def _agent_plan(self, brief: ResearchBrief) -> SearchPlanResult:
        # 多轮迭代（最多 3 轮）
        for iteration in range(self.max_iterations):
            plan = self._invoke_llm(brief)
            hits = self._evaluate_plan(plan)
            if hits >= self.min_hits:
                return plan
        return SearchPlanResult(warnings=["检索规划迭代未达标"])
```

### 3.3 AnalystAgent — Reason-via-Artifacts

**文件**：`src/research/agents/analyst_agent.py`

```python
class AnalystAgent:
    """分析 Agent：Reason-via-Artifacts 流水线"""
    def run(self, brief: ResearchBrief, paper_cards: list[PaperCard]) -> dict:
        # Step 1: 构建 LangGraph RVA 流水线
        graph = self._build_rva_graph()
        initial_state = {
            "topic": brief.topic,
            "paper_cards": paper_cards[:20],  # ⚠️ 截断问题
            "structured_cards": {},
            "comparison_matrix": {},
            "outline": {},
            "draft": {},
        }
        result = graph.invoke(initial_state)
        return result

    def _build_rva_graph(self) -> StateGraph:
        """RVA LangGraph：seed → structured_cards → matrix → outline → draft"""
        from langgraph.graph import StateGraph, START, END
        workflow = StateGraph(ReasoningState)
        workflow.add_node("seed_reasoning_state", self._seed_state)
        workflow.add_node("build_structured_cards", self._build_cards)
        workflow.add_node("build_comparison_matrix", self._build_matrix)
        workflow.add_node("build_outline", self._build_outline)
        workflow.add_node("build_report_draft", self._build_report)
        workflow.add_node("verify_and_finalize", self._verify)
        # 硬编码线性链（无置信度反馈）
        workflow.add_edge(START, "seed_reasoning_state")
        # ... 线性连接 ...
        return workflow.compile()
```

**RVA 流水线设计优势**：
- 5 个独立节点，每个节点专注单一任务
- 中间工件（structured_cards / comparison_matrix / outline）可被后续节点复用
- 支持并行构建 structured_cards（按类别分组）

## 4. Agent 间通信机制

### 4.1 共享状态传递

所有 Agent 通过 `AgentState` TypedDict 共享状态：

```python
class AgentState(TypedDict):
    raw_input: str
    brief: ResearchBrief | None
    search_plan: SearchPlan | None
    rag_result: RagResult | None
    paper_cards: list[PaperCard]
    draft_report: DraftReport | None
    review_feedback: ReviewFeedback | None
    final_report: FinalReport | None
    tokens_used: Annotated[dict, operator.add]    # 累加
    warnings: Annotated[list, operator.add]     # 累加
    node_events: list[dict]                      # SSE 事件
```

**通信模式**：共享内存（Shared Memory），非消息队列。Agent 按顺序执行，通过状态 patch 传递数据。

### 4.2 Handoff Trace

Supervisor 记录每次 Agent 切换的 trace：

```python
class HandoffRecord(TypedDict):
    from_agent: AgentRole
    to_agent: AgentRole
    node: str
    backend_mode: str
    tokens_used: int
    duration_ms: float
    timestamp: str
```

```python
# 改进后的 ReviewerAgent（MAR 模式）
class ReviewerAgent:
    """
    MAR 架构：
    - Actor：生成 ReviewFeedback
    - Diverse Critics：提供多视角批评（覆盖性/Citation/方法论/一致性）
    - Judge：综合批评生成统一反思
    """
    def run(self, draft: DraftReport, paper_cards: list[PaperCard]) -> ReviewFeedback:
        # 多视角批评（并行）
        critics = {
            "coverage": CoverageCritic().critique(draft, paper_cards),
            "citation": CitationCritic().critique(draft),
            "methodology": MethodologyCritic().critique(draft),
            "coherence": CoherenceCritic().critique(draft),
        }

        # Judge 综合：避免单视角偏见
        synthesis = JudgeLLM.synthesize(critics)
        return synthesis
```
